L4 特集号向け追加計算：実行マニュアル
=====================================

収録ファイル
------------
- L4_special_issue_retrieval_geometry_simulation_20260715_0835JST.py
- L4_special_issue_retrieval_geometry_manual_20260715_0835JST.txt

目的
----
次の2点を一括して調べます。

1. 幾何学的なHebbian書き込みコストと、Hopfield検索性能の関係
2. pattern-overlap統計から得られる解析予測と、数値計算の比較

主な観測量
----------
- intensive writing cost F_int
- cumulative writing cost F_cum
- final retrieval overlap
- retrieval success probability
- correlated minus uncorrelated retrieval degradation
- Delta F_int と retrieval degradation のPearson/Spearman相関
- analytical prediction と simulated Delta F_int の対応

必要環境
--------
Python 3.10以降を推奨します。

必要パッケージ:

  python3 -m pip install numpy matplotlib

PDF図は作らず、PNGのみを生成します。
以前のfontTools/xml.etree問題を回避できます。

2. 推奨計算
-----------

  python3 L4_special_issue_retrieval_geometry_simulation_20260715_0835JST.py \
    --outdir L4_special_issue_retrieval_geometry_results \
    --Ns 128 256 512 \
    --loads 0.05 0.10 0.15 0.20 0.30 \
    --generators latent block mixture \
    --corrs 0.15 0.25 0.35 0.45 \
    --cue-noises 0.05 0.10 0.15 0.20 \
    --reps 20 \
    --cues-per-noise 8 \
    --max-steps 20 \
    --lambda-interference 2.0

この設定は、まず傾向を確認するための推奨版です。

---------------

  data/completed_tasks.txt

へ記録します。

計算途中で端末を閉じても、原則として最初からやり直す必要はありません。

9. 結果の判断
-------------

望ましい結果:
- Delta F_int と retrieval degradation に正の相関がある。
- 相関強度が大きいほど、両方が増える。
- latent/blockで解析予測と数値結果が対応する。
- mixtureでも定性的傾向が維持される。

慎重な結果:
- Delta F_intは正だがretrieval degradationとの相関が弱い。

この場合の安全な結論:
  幾何学的コストはHebbian書き込み干渉を測るが、
  検索性能の直接的な予測量とは限らない。

避ける主張:
- 非ゼロの熱力学極限
- retrieval性能改善アルゴリズムを実証した
- Transformer/LLMに直接適用済み
- 任意の相関アンサンブルで普遍的に成立
